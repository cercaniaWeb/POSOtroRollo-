1. Visión Holística del Proyecto

El Otro Rollo POS no es simplemente un software de venta; es un ecosistema operativo diseñado para resolver la fricción tecnológica en entornos de alta demanda y conectividad volátil. Desarrollado específicamente para el Parque Acuático El Otro Rollo en Puebla, este sistema unifica la operación de Restaurante y Tienda bajo una sola interfaz inteligente.

1.1 Filosofía Local-First

A diferencia de los sistemas tradicionales basados en la nube, nuestra arquitectura permite que cada terminal funcione como un nodo independiente. Los datos se persisten localmente y se sincronizan cuando la red está disponible, eliminando las caídas de sistema que afectan los ingresos en horas pico.

3. Manual de Operación Maestro

Guía detallada para el personal operativo y administrativo.

3.1 El Punto de Venta (POS) - "El Corazón"

Diseñado para ser operado con una sola mano en dispositivos táctiles. Permite capturar pedidos masivos en segundos.

  • Filtros Rápidos: Buscador inteligente que filtra por nombre o categoría (Cocina/Tienda).
  • Confirmación de Órdenes: Al confirmar, el pedido se distribuye automáticamente: lo de cocina al KDS y lo de tienda al inventario.

3.2 Kitchen Display System (KDS) - "La Eficiencia"

Monitorea y gestiona el flujo de preparación sin un solo ticket de papel.

  • Estatus de Semáforo: Recibido (Amarillo), En Preparación (Azul), Listo (Verde).
  • Control de Tiempo: Contador visual de minutos transcurridos para asegurar el cumplimiento del SLA de servicio.

3.3 Huéspedes y Carga a Cuenta

Permite una experiencia operativa libre de efectivo (Cashless) mediante el registro de manillas o identificadores.

Desde esta pantalla, el administrador puede realizar "Cargos" rápidos a la cuenta de un huésped y procesar el "Pago Final" al momento del checkout, centralizando toda la deuda del visitante en un solo ticket.

3.4 Inventario y Stock Dual

Gestión inteligente de existencias para dos mundos diferentes.

  • Cocina: Configurado como "Ilimitado" para productos preparados al momento.
  • Tienda: Control unitario estricto. El sistema bloquea la venta automáticamente si el stock llega a cero.

4. Control Financiero y Auditoría

Bóveda de transacciones y herramientas de cierre de turno.

4.1 Historial de Ventas

Cada peso está rastreado. El sistema registra el ID de transacción, el método de pago y el operador que realizó la venta.

4.2 El Corte de Caja

La función más crítica para la transparencia. Al presionar "Corte de Caja", el sistema resume los ingresos en Efectivo vs Tarjeta, genera el reporte final exportable y reinicia los contadores de forma segura para el siguiente operador.
